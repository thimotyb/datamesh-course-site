1 The Consumer Journey: Find, Understand, and Use

The success of a Data Mesh is measured by how effectively consumers can reuse data across domain boundaries. This is achieved through a self-service journey consisting of three primary steps:

  • Find (Discovery): Consumers perform declarative searches in a global catalog to find products that match their business requirements using metadata tags.
  • Understand (Exploration): Using interactive tools like BigQuery Studio or Jupyter Notebooks, consumers experiment with the data to ensure it meets their quality and production needs.
  • Use (Consumption): The final stage is programmatic integration into BI reports, AI/ML models, or real-time operational processes.

2 Discovery Mechanisms: The Central Catalog

A Central Data Catalog acts as the single source of truth for the mesh. It enables "declarative search," allowing consumers to find data based on business logic (e.g., "all mortgage-related products") rather than physical location or technical implementation.

2.1 Role of Dataplex and Metadata Tags

Dataplex facilitates this cataloging by associating rich metadata tags with each product. These tags include critical information such as the Data URI (direct access point), supported consumption modes, and instructions on how to request access. This metadata ensures that data remains discoverable without needing a central team to act as a gatekeeper.

3 Evaluating Data Products: The "Data Product Contract"

Before integrating a product, consumers must evaluate its "contract." This agreement between producer and consumer includes:

  • Documentation: Clear guidance on how to interpret and use the data correctly.
  • Schemas: Explicit structures, particularly vital for event-based systems like Pub/Sub.
  • SLOs/SLAs: The trustworthiness and reliability of the data. If the provided Service Level Agreements are insufficient, consumers use the feedback loop to request improvements.

4 Access Control and Self-Service Workflows

Access is managed via Identity and Access Management (IAM) and VPC Service Controls. A standard pattern is the use of BigQuery Authorized Views, which provide secure access to specific slices of data without exposing raw tables.

Ideally, the request process is a self-service workflow. Data product tags provide the necessary instructions, reducing the time required for a consumer to gain access to a new domain's data.

5 Technical Consumption Patterns

Data products are designed to support multiple technical patterns simultaneously, matching the specific needs of different consumer tools.

Figure M07.01 illustrates this diversity. A single domain exposes four distinct interfaces: two via Authorized Datasets (for BI tools like Looker), one via the Storage Read API (for high-throughput ML training on Dataproc), and a custom Data Access API (for operational apps).

5.1 Choosing the Right Interface: SQL vs. API vs. Streams

  • BigQuery (SQL): The primary choice for BI and analytical queries.
  • BigLake: Unified access to data in Cloud Storage and other clouds with BigQuery security.
  • Storage Read API: High-speed access for Data Science and large batch processing.
  • Pub/Sub: Near-real-time event streams for reactive operational processes (e.g., fraud detection).

6 The Feedback Loop and Center of Excellence (COE)

Communication is vital for a healthy mesh. If a product is missing or its interface is insufficient, the consumer contacts the producer. If the responsible domain is unclear, the **Data Mesh Center of Excellence (COE)** helps determine the best domain to produce the required asset. This dynamic loop ensures the mesh evolves according to the organization's real business needs.

Key takeaways

  • Consumption is a self-service journey of discovery, exploration, and integration.
  • A global catalog with rich metadata tags is essential for finding relevant data products.
  • Consumers evaluate products based on explicit contracts (schemas, documentation, SLOs).
  • Standard patterns like Authorized Views and the Storage Read API ensure secure and efficient access.
  • A continuous feedback loop between consumers and producers drives the mesh's evolution.
