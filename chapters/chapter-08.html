1 The Role of Dataplex Universal Catalog

Dataplex Universal Catalog is a fully managed, automated metadata management service designed to create a single source of truth for distributed data and AI assets. Its primary purpose is to automate discovery and eliminate the need for manual data inventories.

By providing a unified knowledge base, the catalog ensures that analytics and machine learning models are built on trusted, compliant information. It solves the "cold start" problem for data consumers, allowing them to find, understand, and verify the quality of data products without needing constant support from engineering teams.

2 Metadata Collection Mechanisms

The catalog operates on a hybrid model that combines the efficiency of automation with the richness of business context.

2.1 Automated Harvesting vs. Manual Enrichment

  • Automated Harvesting: Dataplex automatically ingests technical metadata, such as schemas, table names, and partition details, from Google Cloud sources like BigQuery, Cloud Storage, and Pub/Sub.
  • Manual Enrichment: Data owners can manually add business context, such as data quality scores, ownership information (email, team name), and business glossary terms. This makes technical assets meaningful to non-technical users.
  • Metadata Feeds: Near real-time change feeds allow organizations to trigger automated workflows, such as security audits, whenever a metadata change is detected.

3 Supported Resource Types: Data and AI

Dataplex indexes a vast array of resources across the Google Cloud ecosystem:

  • Analytics: BigQuery datasets, tables, views, and BigLake tables.
  • Databases: AlloyDB, Cloud SQL, Spanner, and Bigtable.
  • AI/ML: Vertex AI models, datasets, and feature groups.
  • Lakehouse: Dataproc Metastore services and files stored in Cloud Storage.
  • Custom Sources: Metadata from third-party systems or on-premises databases can be imported using managed connectivity pipelines.

4 Advanced Data Discovery Features

Discovery is powered by a robust search engine that respects organizational security boundaries.

  • Faceted Search: Users can filter results by resource type, project, physical location, or custom metadata fields (e.g., "show all high-quality tables in the US region").
  • Search Syntax: Supports advanced predicates (e.g., `type=table name:sales`) and logical operators for precise discovery.
  • Security-Aware Search: Search results are strictly limited by **IAM roles** and **VPC Service Controls**. Users only see metadata for assets they are authorized to access.

5 Metadata Modeling: Entries and Aspects

Dataplex uses a flexible, typed metamodel to organize and standardize information across the organization.

  • Entry: The atomic unit representing a specific data asset, such as a BigQuery table or a Vertex AI model.
  • Aspect: A set of related metadata fields (e.g., "Contact Information") that can be attached to an entry to describe its characteristics.
  • Aspect Type: A reusable template that defines the structure of an aspect, ensuring consistency across different domains.

5.1 Defining Metadata Standards with Entry Types

Organizations can use Entry Types to enforce metadata standards. For example, a "Standard Data Product" type can be defined to require specific aspects, such as ownership and data quality metrics, before an asset can be published to the catalog. This ensures that the mesh remains trustworthy and governed.

6 Enabling Data Mesh Governance

In a Data Mesh, Dataplex Universal Catalog acts as the Mesh Supervision Plane. It enables federated discovery by allowing domains to publish their products in a consistent format while maintaining local autonomy. It also tracks data lineage, helping organizations understand how data flows and transforms across domain boundaries, which is critical for audit and compliance.

Key takeaways

  • Dataplex Universal Catalog is the central discovery and governance engine for the data mesh.
  • It automatically harvests technical metadata while allowing manual business context enrichment.
  • Supports a wide range of assets, from BigQuery tables to Vertex AI machine learning models.
  • Metadata modeling using Entries and Aspects ensures consistent standards across domains.
  • Search functionality is secure-by-default, respecting IAM and VPC Service Controls.
