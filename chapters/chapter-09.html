1 Metadata Enrichment: Aspects and Aspect Types

Metadata enrichment is the process of adding business context and operational details to technical data assets. In Dataplex, this is achieved through a structured model consisting of Aspect Types and Aspects.

  • Aspect Type: A reusable JSON template that defines the structure of metadata fields. It acts as a schema for the information you want to capture (e.g., a "Data Compliance" template).
  • Aspect: A specific instance of an Aspect Type attached to a data entry. It contains the actual values (e.g., "PII Classification: High").

1.1 Custom vs. System Aspect Types

Dataplex provides two categories of templates:

  • System Aspect Types: Pre-defined by Google to capture standard information like schemas, contact details, and data profiles. These are managed automatically.
  • Custom Aspect Types: Defined by the organization to meet specific business needs. They can be Global (available across all regions) or Regional (restricted to specific geographical areas to comply with data residency rules).

2 The Process of Enriching Entries

Enrichment allows domain teams to transform raw technical entries into discoverable business products. Aspects are considered integral parts of the Entry resource; modifying an aspect involves updating the entry itself.

2.1 Defining Metadata Fields and Complex Records

Aspect Types support a variety of data types, including Text, Enum, Array, Map, and Record. For complex metadata, Dataplex uses Record references, allowing for nested objects and recursive definitions without duplicating field structures. Enrichment can be applied at the Entry-level (describing the whole table) or the Column-level (describing specific fields within the schema).

3 Entry Types: System vs. Custom

Every asset in the catalog is governed by an Entry Type template that defines which aspects are required.

  • System Entry Types: Automatically created for Google Cloud resources (e.g., BigQuery tables). Required aspects like schemas are read-only for users.
  • Custom Entry Types: Created for assets from external sources (e.g., on-premises databases). Users have full control over the required aspects, which must be populated when the entry is created.

4 Ingesting Metadata from Custom Sources

To build a truly universal catalog, organizations must ingest metadata from sources outside of Google Cloud. Dataplex supports two main ingestion patterns to bring this data into the mesh.

4.1 Managed vs. Custom Connectivity Patterns

  • Managed Connectivity: Uses built-in Dataplex capabilities to automatically sync metadata from supported third-party environments.
  • Custom Connectivity: Involves building custom pipelines or using Google Cloud Workflows to push metadata into the Universal Catalog via APIs. This is used for proprietary systems or sources without native connectors.

5 OpenLineage and Third-Party Integrations

Dataplex integrates with the OpenLineage standard to track data movement and transformations across diverse systems. This enables organizations to maintain a unified view of lineage, integrating metadata from third-party tools like Informatica, Collibra, or custom-built processing engines into a single supervision plane.

6 Managing the Metadata Lifecycle

A healthy catalog requires automated lifecycle management. Organizations can use Metadata Change Feeds (via Pub/Sub) to receive real-time notifications when entries or aspects are modified. These feeds can trigger automated actions, such as re-running a data quality scan or synchronizing changes back to an external governance tool, ensuring the catalog remains accurate and trustworthy.

Key takeaways

  • Aspect Types act as reusable schemas for capturing business and operational metadata.
  • Enrichment can be applied at both the entry level and the granular column level.
  • Custom Entry Types enable the cataloging of assets from any source, including on-premises.
  • OpenLineage support provides a standardized way to track data flow across the enterprise.
  • Automated lifecycle management via Change Feeds ensures metadata consistency and trust.
